# js模块化介绍

#### 精简回答：

1.模块化的概念：将一些复杂的程序依据规范封装成一些小块，并组合在一起；块内部数据是私有的，只向外部暴露一些接口与其他模块通信

2.模块化的作用：避免变量冲突，便于代码编写与维护

3.模块化的历程：


- 普通写法：（将变量或函数放置在全局）变量冲突容易污染

```js
var name = '雾里天';
function fun1(){
    name = '雾里天1';
}
function fun2(){
    name = '雾里天2';
}
```
- 对象封装：（将变量或函数放在对象内）这种方法虽然解决了变量冲突问题，但是容易被外部随意修改

```js
let funAll = {
    name:'雾里天',
    fun1:function(){
        var name = '雾里天1';
        console.log(name);
    },
    fun2:function(){
        var name = '雾里天2';
        console.log(name);
    }
}
funAll.name;// 雾里天
funAll.fun1();// 雾里天1
funAll.fun2();// 雾里天2
```

- 匿名函数：（将变量或函数放置在匿名函数中）解决了函数污染及变量随意被修改问题是模块化规范的基石

4.模块化方案

- CommonJS

    1.在node环境中使用，不支持浏览器环境
    
    2.使用require引入依赖
    
    3.使用exports暴露模块
    
- AMD

    1.浏览器环境下的异步加载模块
    
    2.RequireJS遵循的规范
    
    3.AMD 推崇依赖前置

- CMD

    1.浏览器环境下，同时支持异步和同步加载
    
    2.SeaJS遵循的规范
    
    3.CMD 推崇依赖就近

- ES6 module

    1.浏览器环境、服务器环境都支持
    
    2.编译时就能确定模块的依赖关系及变量，其他模块规范都是在运行时确定的
    
    3.export命令用于规定模块的对外接口

    4.import命令用于输入其他模块提供的功能
